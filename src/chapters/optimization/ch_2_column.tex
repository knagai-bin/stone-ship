社会便益を最大化するバスの最適ルートを探索するためのMFDを基礎としたヒューリスティクス手法の比較と，ミクロへの応用について研究を行った．
用いたデータは2007年松山都市圏パーソントリップ調査の個票データ（出発時刻が7時から9時で代表交通利用手段の分類が自動車のもの）と，松山市全域（主要な道路，すべて1車線と仮定）と市街地（自動車通行可能なリンク，車線数考慮）の2種類の道路ネットワークを用いた．

その後，各ゾーンの総車線距離を計算し，そこに車の限界密度（1キロに120台）を設定して乗じることで，ゾーンごとの最大滞在可能台数を算出した．また，各ゾーン内での動きをゾーン内完結，ゾーン出発，ゾーン到着，ゾーン通過の4つに分類しそれぞれの平均最短経路距離を算出した．

次に，これらを元に，実際の混雑状況の再現を試みた．具体的には，ゾーン内にいる車両数と最大滞在可能台数の比率から，
\begin{equation}
  V = \frac{V_f}{1 + \alpha (n / N_{\max})^\beta}
  \label{eq:bpr_function}
\end{equation}
という式\eqref{eq:bpr_function}（BPR関数）を用いてそのゾーンでの速度を導出する．ここで，$V$ はそのゾーンでの実際の速度，$V_f$ は自由旅行速度（混雑がない状態での速度），$n$ はゾーン内にいる現在の車両数，$N_{\max}$ はゾーンの最大滞在可能台数（交通容量），$\alpha$ および $\beta$ は交通特性を表すパラメータである．

先に求めた平均最短経路距離をこの速度 $V$ で割ることで，そのゾーンの通過に何分かかるのかを計算する．このプロセスを7:00から9:59まで1分おきで行い，各ゾーンの通過にかかる時間を書き換え，その1分に発生する車（PTの時間平均とポアソン分布を用いて決定）の最短経路計算に用いることで，「車が増える $\to$ 速度が落ちる $\to$ 後から来た車が別のルートを選ぶ（ダイクストラ法） $\to$ 車が移動して別のゾーンが混む」というダイナミクスを表現し，1分ごとの各状態でどのゾーンに何台の車がいたのかを記録した．
以上の結果を用いて最適なバスのルートの最適化を考える．目的関数には，社会的便益を設定した．具体的には，新設される10ゾーンを巡回する環状バス路線に対し，各ODの移動者が自動車とバスのどちらを選択するかを，所要時間と費用を変数としたロジットモデルを用いて確率的に推定する．バスへの転換によって得られる利用者全体のログサム便益から，バスの運行コストを差し引いた社会的便益 $SW$ は次式で定義される．

\begin{equation}
  SW = \sum_{od} \left( \frac{\ln(\exp(V_{car}) + \exp(V_{bus})) - \ln(\exp(V_{car}))}{|\beta_{time}|} \cdot VOT \cdot D_{od} \right) - C_{op} \cdot T
  \label{eq:social_welfare}
\end{equation}

ここで，$V_{car}, V_{bus}$ はそれぞれ自動車とバスの効用関数，$\beta_{time}$ は時間パラメータ，$VOT$ は時間価値，$D_{od}$ はOD間の交通需要，$C_{op}$ は単位時間あたりの運行コスト，$T$ はバスの総運行時間を表す．本研究ではこの $SW$ を目的関数とし，最適化のスコアとしている．

また制約条件としては以下を組み込んだ．
\begin{itemize}
    \item \textbf{ルート長の固定:} 短すぎるルートでは意味をなさないので，10ゾーンを巡るように固定した．
    \item \textbf{同一ゾーンの重複禁止:} 評価関数の冒頭では同一ゾーンを通っているものに極端なマイナス効用を与え，探索候補から除外されるように設計するとともに，GAは交差時に重複したノードはルート上にない他のゾーンから新しいものを選ぶようにし，SAとTSはReplaceの構造上重複が起きないようにしている．
    \item \textbf{バスの輸送量制約:} 各区間の乗客数を集計し，最も混む区間の乗客数が450人を超えた場合は450人までしかバスに乗せないように実装している．
    \item \textbf{経路の連続性とペナルティ:} 空間的に隣接していないゾーン間をワープするような非現実的なルート生成を防ぐため，隣接していないゾーン間のつながりに対しては物理的距離に基づく迂回ペナルティを課した．
    \item \textbf{輸送容量の制約:} バスの最大定員を設定し，特定区間に需要が集中して定員を超過した場合は，超過分の転換行動を無効化することで便益の過大評価を防いだ．
    \item \textbf{混雑緩和のフィードバック（静的均衡へのすり合わせ）:} 先の動的シミュレーションで得られた平均滞在台数を初期の背景交通量として設定した．バスへの転換によって自動車の交通量が減少すると，BPR関数に従い道路の走行速度が向上する．速度が向上すると自動車の利便性が回復し，バスへの転換率が再び低下するという需要と供給の反復が発生する．本研究ではこの再計算を3回反復させている（前述の動的な交通配分のシミュレーションを毎回実行する計算コストを回避するため，ルート選択は無視し，モード選択にのみ注目している点に注意）．
\end{itemize}

上記の都市全体のゾーンから最適な10ゾーンの組み合わせと巡回順序を決定する問題は，膨大な探索空間を持つ組み合わせ最適化問題である．そのため本研究では，性質の異なる3種類のメタヒューリスティクス手法（GA，SA，TS）を実装し，探索性能と計算時間の比較を行った．公平な比較のため，各手法の評価回数が約10,000回となるようにパラメータを調整した（計算時間は環境に依存するため）．各手法のパラメータについて表\ref{tab:hyperparameters}に示す．

\textbf{基礎概念：近傍生成}\\
タブーサーチと焼きなまし法において，探索の要となるのが近傍生成の設計である．本モデルでは以下の2つの操作を組み合わせて新しいルートの候補を作成した．
\begin{itemize}
    \item Swap（入れ替え）: ルート内の2つのゾーンの順番を入れ替える（例: A-B-C $\to$ A-C-B）．通過するエリアは変えずに，バスの巡回順序を変更する．
    \item Replace（差し替え）: ルート内の1つのゾーンを捨てて，まだ使われていない新しいゾーンを組み込む（例: A-B-C $\to$ A-B-Z）．ルートが通る「エリア」自体を変更する．
\end{itemize}
各手法は，この操作を活用して最適解へと迫っている．

\textbf{遺伝的アルゴリズム（GA: Genetic Algorithm）}\\
100個体を保持し，100世代にわたり以下の操作を繰り返す．
交差（Crossover）: スコアの高かった優秀なルートAと，ルートBを選び，あるポイントで分割して繋ぎ合わせる．
修復（Repair）: 繋ぎ合わせた結果，同じゾーンを2回通ってしまうルール違反が起きた場合，重複したゾーンを削除し，まだ使っていないゾーンからランダムに引っ張ってきて穴埋めを行う．
突然変異（Mutation）: 世代交代の際，20\%の確率で個体に対してSwapやReplaceを強制的に行い，新しい変化をもたらすことで，多様性の喪失を防ぐ．

\textbf{焼きなまし法（SA: Simulated Annealing）}\\
1ステップでSwapかReplaceをランダムに行い，1つの近傍解を作成する．序盤はスコアが悪化する「改悪解」も$\exp(\Delta / T)$ という確率で許容することで局所最適解に陥ることを防いだ．徐々に冷却率に従って温度を下げることで，最終的に最適な環状ルートへと収束させた．

\textbf{タブーサーチ（TS: Tabu Search）}\\
現在のルートの近傍を，Swap か Replace をランダムに行うことで毎ステップ20個生成し，最良のものへ移動する局所探索を行った．その際，直近の改変履歴をタブーリストに保持し，次のステップで同じ操作を禁止することで探索のループを防ぎ，効率的に未探索エリアを開拓させた．

性質の異なる上記の3つのメタヒューリスティクス手法（GA，SA，TS）について，それぞれ20回の独立した試行を行い，「社会的便益（スコア）」と「実行時間」の比較を行った結果が図\ref{fig:comparison_results_noloop}および図\ref{fig:comparison_results_loop}である．バスの運行形態として直線ルート（往復なし）を想定した初期モデルと，実用性を考慮して始点と終点を結んだ循環ルートを想定した提案モデルの2パターンで評価を実施した．

結果を比較すると，ルートの前提条件によって最適な探索アルゴリズムが変化するという興味深い結果が得られた．直線ルート想定時には，TSが最も高いスコアを安定して記録した．TSは過去の改変履歴を記憶して重複探索を防ぐため，単純な一直線のルートを無駄なく効率的に探索し当てることに優れていたと考えられる．
一方，本研究の最終的な提案モデルである循環ルート想定時においては，SAが最も高い便益（中央値 約169,000円，最大値 170,000円超）を記録した．循環ルートは「始点と終点が繋がる」という制約が加わるため，探索空間の地形がより複雑になり，局所最適解に陥りやすくなる．SAの「序盤はスコアが悪化する変更も確率的に許容する」という物理的特性が，中途半端な局所解からの脱出を促し，最も広域の需要を拾える大域的最適解へ到達できたものと推察される．

同条件におけるTSは，箱ひげ図の四分位範囲が非常に狭く，安定して高い解を出力する堅牢さを見せたものの，絶対的な最大値や中央値ではSAに及ばなかった．これは，TSが局所探索に優れる反面，循環ルートのような複雑な制約下では，タブーリストだけでは乗り越えられない大きな谷に直面し，そこそこの良いルートで探索が停滞してしまったためと考えられる．

GAは，両条件においてスコアの中央値が最も低く，試行ごとのばらつきも非常に大きかった．ルートの一部を入れ替える交叉を行う際，同じゾーンを2回通るルール違反が頻発し，それをランダムなゾーンで強制修復する処理が介入したことで，世代ごとの優秀なルートの継承が阻害されたためであると考えられる．強制修復の過程で需要量と距離などを反映させて探索することで改善できるとも考えられる．

また，実行時間のグラフからは，評価回数を揃えたことによる妥当性も確認できる．
循環ルートにおけるSAとTSは，どちらも実行時間が約60秒で完全に横並びとなっている．これは，両者が全く同じ回数（10,000回）だけ重たい交通均衡フィードバックを実行したことを示しており，アルゴリズム間の純粋な「1万回の探索の質」の比較が担保されていることの証明である．
GAの実行時間が約46秒とやや短いのは，交差や突然変異によって生じた「重複ルート」が評価関数の冒頭で即座に弾かれるケースが含まれているためであり，実質的な有効探索回数が他の2手法より少なかったことも，スコア低迷の一因と考えられる．

次にここまでで得られたマクロな最適ゾーン経路（回廊）を基敷とし，実際の交差点（ノード）と道路リンクからなる松山中心市街地のミクロレベルの道路ネットワークへの適用と，最終的なバス路線の再最適化を行った．

まず，マクロ最適化で得られたゾーン単位の経路を，実際の道路ネットワークへと落とし込んだ．具体的には，PT調査のゾーンポリゴンとミクロな道路のノードの空間結合を実施した（図\ref{fig:micro_zones}）．

各ゾーン内において，自動車が通行可能なリンクに接続しており，かつ最も接続リンク数の多いノードを代表ノードとして抽出し，マクロな経路を具体的な交差点の移動へと変換した．

ミクロレベルの評価においては，都市部の複雑な交通動態を反映するため，評価モデルに以下の精緻化を加えた．
\begin{itemize}
    \item \textbf{実ネットワークでの渋滞考慮:} 各道路リンクの車線数から物理的な交通容量を算出し，実際の背景交通量データに基づくBPR関数を適用した．これにより，幹線道路と生活道路のキャパシティの違いや，交差点間における慢性的な渋滞による遅延をシミュレーションに組み込んだ．
    \item \textbf{最低運行距離制約の導入:} ミクロ空間において厳密な運行コスト（1分あたりの費用）を計上すると，探索アルゴリズムがコストを削るために，極端に近い交差点だけを巡回してすぐに車庫へ帰るという自明な局所解へ収束してしまう問題が生じる．これを防ぐため，最低運行距離制約を設け，それに満たないルートにはペナルティを課す設計とした．
\end{itemize}

ミクロレベルでの経路最適化には，マクロ評価で最も堅牢な性能を示したタブーサーチ（TS）を再度採用した．ミクロな道路網においては，GAの交叉処理（ルートの分断と結合）を用いると，物理的に走行不可能な経路や，同じ交差点を何度も往復するルール違反が大量発生し，探索効率が著しく低下してしまった．そのため，ルート内の交差点順序の入れ替え（Swap）と，近隣の別交差点への差し替え（Replace）による局所探索を反復する，階層型のTSモデルを構築した．

ミクロ道路網特有の性質と，階層型アプローチの有効性を検証するため，以下の4種類の初期解生成アプローチを用意し，十分な探索回数（300イテレーション）のTS探索を行うモンテカルロ評価（各10回試行）を実施した．
\begin{enumerate}
    \item \textbf{Macro-TS（提案手法）:} マクロ最適解として選ばれたゾーンの代表ノードから探索を開始する．
    \item \textbf{Greedy（貪欲法）:} 最も需要の高い交差点を起点とし，そこから物理的に最も近い交差点を順に繋いだ経路から開始する．
    \item \textbf{Demand-Focus（需要特化）:} 距離を問わず，単純に需要が集中している上位の交差点を結んだ経路から開始する．
    \item \textbf{Random（ランダム）:} 完全ランダムに選ばれた交差点の組み合わせから開始する．
\end{enumerate}

十分な探索回数を与えることで，ミクロ道路網内の渋滞や迂回を避け，社会的便益がプラスとなる実用的なバスルートが複数発見された．しかし，各手法の成績を比較すると，事前の予測に反して非常に興味深い知見が得られた．

第一に，平均スコアおよび勝率においてRandomが最も優れた成績を収めた点である．第二に，マクロ最適解を用いた「Macro-TS」が，唯一黒字化を一度も達成できず，赤字の局所最適解から抜け出せなかった点である．

この逆転現象は，マクロ最適解が抱えるミクロ空間の呪縛を強く示唆している．マクロレベルで有望とされたゾーン群は，需要が極めて高い反面，実際の道路網で見ると信号が多く，慢性的な渋滞が発生し，迂回を強いられる中心市街地の深部に該当することが多い．Macro-TSは，その渋滞エリアのど真ん中に初期解が固定されてしまうため，TSをもってしても根本的な遅延による運行コスト増大から抜け出せなかったと考えられる．

一方で，ランダム初期化は都市全体に散らばった状態から探索をスタートするため，初期解の多様性が担保される．TSの強力な局所探索能力と十分な探索回数を組み合わせることで，特定の渋滞エリアに過度に固執することなく，走りやすさと需要のバランスが取れた現実的な抜け道を柔軟に開拓できたことが，安定した高スコアに繋がったと推察される．なお，GreedyおよびDemand-Focusは，全試行の中で最高益を叩き出すポテンシャルを見せたものの，初期配置によっては行き止まりや大迂回の罠にハマり大赤字を出すハイリスク・ハイリターンな挙動を示した．

本研究の階層型シミュレーションを通じ，マクロな直線距離ベースの最適化は有望な需要回廊の発見に寄与するものの，それをそのままミクロな道路網へ適用すると，局所的な道路構造や渋滞の影響で運行コストが想定以上に増大し，最適解から遠ざかることが判明した．現実のマイクロトランジット路線設計においては，需要の高いエリアをピンポイントで狙う戦略（マクロ最適解や需要特化型）に依存するのではなく，完全ランダム初期化による多様性を確保したマルチスタート型タブーサーチを用いることが，複雑な都市の道路網において高収益かつ安定したバスルートを構築する上で極めて有効であるのではないかと考えた．また，初期値によって分散が大きいものについては計算回数を増やすよりもさまざまな初期値から探索を行うことが有効であるのではないかとも考えられた．
